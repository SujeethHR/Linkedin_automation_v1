# LinkedIn AI Post Studio — Product Requirements Document

**Version:** 2.0  
**Last Updated:** March 2026  
**Status:** Active Development

---

## 1. Overview

LinkedIn AI Post Studio is a local Python web application that automates personal brand content creation and publishing on LinkedIn. It aggregates fresh industry news across multiple domains, uses an LLM to draft contextually relevant posts, and publishes them directly to LinkedIn via the official API — with scheduling, image attachment, and calendar management built in.

The app runs entirely on the user's machine (`python app.py`) and is accessed via browser at `http://localhost:5001`. No cloud hosting or external database is required.

---

## 2. Problem Statement

Maintaining a consistent LinkedIn presence requires:
- Staying on top of fast-moving news across multiple domains simultaneously
- Writing engaging posts in a consistent voice — repeatedly, multiple times per week
- Scheduling posts at optimal times without manual effort
- Tracking what has already been posted to avoid duplication

Doing this manually is time-consuming and inconsistent. This tool collapses the full workflow — research → draft → review → schedule → publish — into a single interface.

## 4. Core Features

### 4.1 Trend Fetching (Step 1)
- Fetches fresh news from the past 7 days using DuckDuckGo web search + 50+ curated RSS feeds
- Optional: Google Trends integration (rising queries by region) and Twitter/X public posts via DDG
- Domain-aware: selecting a category pill automatically routes to the correct RSS feeds and search queries
- Seen-articles cache: articles already shown are suppressed for 30 days; user can clear the cache at any time
- Output: list of topic cards showing headline, summary, relevance note, and heat level (hot / rising / new)

### 4.2 Topic Selection (Step 2)
- User selects which topics to draft posts for (multi-select cards)
- Topics include source, heat indicator, and a "why it matters" note generated by the LLM

### 4.3 Post Review & Editing (Step 3)
- LLM drafts a post for each selected topic
- User can edit the text inline before approving
- AI-generated hashtag pills — click to append to the post
- Manual image upload: drag & drop or file picker → uploaded to LinkedIn Assets API → attached to post
- Per-post verdict: Approve / Schedule (with datetime picker) / Reject
- Regenerate button: re-drafts a post without refetching trends

### 4.4 Publishing (Step 4)
- Publishes all approved posts to LinkedIn in one click
- Supports text-only and image+text posts
- Duplicate guard: posts already published by the scheduler are skipped
- Post URNs are stored in `schedule.json` for analytics tracking

### 4.5 Calendar Tab
- Monthly grid showing scheduled, published, and failed posts as colour-coded dots
- Actions per post: Publish now / Reschedule / Delete

### 4.6 Custom Topic Tab
- User types any topic → app scrapes news + ArXiv research papers
- Selects sources → LLM drafts a post based on chosen source material
- Supports angle/focus customisation and all tone/length options

### 4.7 Analytics Tab
- Fetches post performance data via LinkedIn `memberCreatorPostAnalytics` API
- Metrics: impressions, clicks, likes, comments, shares, engagement rate
- Single post lookup (by post URN) or bulk fetch for all published posts
- **Note:** Requires `r_member_social` OAuth scope — only available to LinkedIn Marketing Developer Platform partners. Standard developer accounts have `w_member_social` only.

### 4.8 Debug Tool
- Validates LinkedIn token and URN before publishing
- Test-publishes a post and reports the exact failure reason if it fails

## 6. Technical Architecture

### Stack
- **Backend:** Python 3.10+, Flask 3.x
- **Frontend:** Single-page HTML/CSS/JS (no framework, no build step)
- **LLM:** Abacus.AI RouteLLM API (OpenAI-compatible `/v1/chat/completions`)
- **Web search:** DuckDuckGo via `ddgs` library (no API key)
- **RSS parsing:** Python standard library `xml.etree.ElementTree`
- **Scheduler:** APScheduler (background thread, checks every 60 seconds)
- **LinkedIn API:** `/v2/ugcPosts` (publish), `/v2/assets` (image upload), `/rest/memberCreatorPostAnalytics` (analytics)

### Data Storage
| File | Contents |
|------|----------|
| `schedule.json` | All posts with status, timestamps, post URNs, errors |
| `seen_articles.json` | Article titles and URLs with timestamps (30-day TTL) |

Both files are created automatically on first run in the working directory.

### LLM Calls Per Session
| Task | Max Tokens |
|------|-----------|
| Trend extraction | 3000 |
| Post drafting | 1500 |
| Hashtag generation | 80 |
| Custom topic drafting | 1500 |

## 7. Configuration (.env)

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `ABACUS_API_KEY` | ✅ Yes | — | Abacus.AI API key |
| `ABACUS_BASE_URL` | No | `https://routellm.abacus.ai/v1` | RouteLLM endpoint |

| `LINKEDIN_URN` | ✅ Yes | — | `urn:li:person:XXXXXXXX` |

| Method | Endpoint | Description |
|--------|----------|-------------|

| POST | `/api/debug-publish` | Validate credentials + test publish |
| POST | `/api/post-analytics` | Single post analytics |
| POST | `/api/all-post-analytics` | All published posts analytics |

| POST | `/api/reschedule/<id>` | Change post scheduled time |

## 10. Known Limitations

| Limitation | Reason |
|-----------|--------|
| Analytics requires `r_member_social` | LinkedIn restricts this to Marketing Developer Platform approved apps |
| Token expires every 60 days | LinkedIn policy — must be regenerated manually |
| AI image generation removed | All external image generation APIs now require paid subscriptions |
| Google Trends optional | `pytrends` can be unreliable; disabled by default |

---

## 11. Out of Scope

- AI-generated images
- Multi-account LinkedIn support
- Company page posting
- Analytics charts / visualisations
- Mobile app or hosted/SaaS version
